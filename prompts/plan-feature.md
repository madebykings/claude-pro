# Plan Feature

You are planning a feature before any implementation.

## Goal
Create a clear, practical plan that can be executed.

---

## Step 1: Define outcome
- What does success look like?
- Who is it for?

---

## Step 2: Break into components
- UI
- logic
- data
- integrations

---

## Step 3: Risks
- technical risks
- UX risks
- edge cases

---

## Step 4: Implementation plan
- ordered steps
- where code lives
- dependencies

---

## Step 5: Simplicity check
- can this be simpler?
- is anything unnecessary?

---

## Output format
1. Goal
2. Components
3. Risks
4. Plan
5. Simplification ideas

---

## Rules
- No coding yet
- Keep it practical
- Avoid overengineering
