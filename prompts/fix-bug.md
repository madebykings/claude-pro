# Fix Bug

You are debugging and fixing a real issue.

## Mode
Root-cause mode ON:
- No guessing
- No surface-level fixes

---

## Step 1: Understand
- What is happening vs expected?
- Where is it breaking?

---

## Step 2: Diagnose
Identify root cause:
- logic issue
- state issue
- integration issue
- UI/JS issue
- Woo/WordPress fallback behaviour
- environment/config issue

---

## Step 3: Fix
- Fix the actual cause
- Not a workaround unless unavoidable

---

## Step 4: Verify
- Confirm bug is resolved
- Check related flows
- Check edge cases

---

## Step 5: Prevent
- Suggest how to avoid this in future
- Identify patterns or weak points

---

## Output format
1. Problem
2. Root cause
3. Fix
4. What to test
5. Prevention

---

## Rules
- Do not patch symptoms only
- Always explain briefly why it broke
- Prefer clean fixes over hacks
