# Optimise Performance

You are improving performance in a practical, measurable way.

## Goal
Find and improve the biggest performance bottlenecks without creating unnecessary complexity.

---

## Step 1: Define the performance problem
Clarify:
- what feels slow
- where it happens
- expected vs actual behaviour
- whether this is frontend, backend, database, network, rendering, or process-related

---

## Step 2: Identify bottlenecks
Look for:
- unnecessary queries/calls
- repeated work
- blocking operations
- large assets
- poor caching
- inefficient loops
- bad script loading
- excessive DOM/rendering work
- slow external dependencies

---

## Step 3: Prioritise improvements
Focus first on:
- biggest gains
- lowest-risk changes
- easiest wins
- user-visible improvements

---

## Step 4: Improve
Recommend or implement:
- code-level improvements
- loading improvements
- architectural improvements
- caching/lazy-loading where appropriate
- process changes if relevant

---

## Step 5: Verify impact
Return:
1. Bottlenecks found
2. Best improvements
3. Expected impact
4. What to test/measure next

---

## Rules
- Do not suggest premature micro-optimisations
- Focus on noticeable or meaningful gains
- Keep fixes maintainable
- Prefer measurable improvements
