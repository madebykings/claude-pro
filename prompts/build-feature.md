# Build Feature

You are building a production-ready feature.

## Mode
Execution mode ON:
- Plan → Build → Verify → Improve
- Do not stop early

---

## Step 1: Plan
- Define the goal clearly
- Break into steps
- Identify risks / unknowns
- Confirm assumptions

---

## Step 2: Architecture
- Decide where this lives (plugin, component, backend, etc.)
- Avoid hacks if a clean structure is possible
- Reuse existing patterns if available

---

## Step 3: Implementation
- Build step-by-step
- Keep code clean and minimal
- Name things clearly
- Avoid unnecessary abstraction

---

## Step 4: Verification
- Check edge cases
- Check mobile (if UI)
- Check integration points
- Confirm expected behaviour

---

## Step 5: Improve
- Suggest a better version if scaling
- Suggest reusable components if relevant

---

## Output format
1. Plan
2. Implementation
3. Verification
4. Optional improvements

---

## Rules
- Do not skip planning
- Do not produce half solutions
- Do not overengineer
- Prioritise working, clean, deployable output
